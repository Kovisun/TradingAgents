# 后续待实现功能

## 功能一：全市场股票筛选（Stock Screener）

**解决的问题**："不知道买什么股"

**功能描述**：
按多维度条件批量扫描全市场 A 股，输出候选股票列表及筛选理由。

**筛选维度**：
- 技术面：MACD 金叉、放量突破、均线多头排列、RSI 超卖/超买
- 资金面：北向资金持续流入、龙虎榜机构净买入、主力资金净流入排行
- 基本面：PE 处于历史低分位、ROE > 15%、营收/利润增长
- 概念热度：近期热门概念板块龙头股
- 综合评分：多因子加权排名

**交互方式**：
- 预设场景模板："短线放量突破"、"北向资金加仓"、"超跌反弹"、"机构密集调研"
- 自定义筛选条件组合
- 筛选结果支持一键发送到分析模块做深度分析

## 功能二：持仓自动跟踪预警

**解决的问题**："持有的股票何时卖，下跌或盈利时该不该卖"

**功能描述**：
对用户持仓股票进行持续跟踪监控，触发条件时给出卖出/持有建议。

**跟踪维度**：
- **价格预警**：跌破止损价、达到目标价、日内跌幅超过阈值
- **技术信号**：MACD 死叉、跌破均线支撑、放量下跌
- **资金面**：北向资金持续流出、龙虎榜卖方席位出现
- **基本面**：业绩预告/快报不及预期、解禁来临、大股东减持
- **综合决策**：每日自动跑一次轻量分析，更新买卖建议

**交互方式**：
- 跟踪看板：一目了然看到所有持仓的最新建议
- 每日开盘前自动分析持仓股票
- 支持设置个股的止损价/目标价

---

## 优先级

1. 股票筛选（先解决选股问题）
2. 持仓跟踪预警（再解决卖出问题）

## 功能三：报告复盘（待实现）

**解决的问题**：对已生成的研报，对比分析日之后的实际走势，验证判断是否正确。

**功能描述**：
在历史报告列表的每条报告后加"复盘"按钮，点击后：
1. 读取报告的决策方向（买入/持有/卖出）和目标价/止损价
2. 拉取分析日后 N 个交易日的实际收盘价
3. 对比：实际走势是否符合预期？
4. 输出：正确/错误、实际涨跌幅、最大回撤

**交互方式**：
- 报告列表每行加复盘按钮
- 点击弹出复盘结果卡片

## 功能四：轻量交易引擎（独立容器，待实现）

**架构定位**：
```
TradingAgents（决策层）──→ 交易引擎（执行层）──→ 券商接口
                              │
                              ├→ 实时行情监控
                              ├→ 毫秒级下单
                              ├→ 风控止损
                              └→ 通知（企微/QQ/飞书）
```

TradingAgents 负责"想"，交易引擎负责"做"，各自独立容器部署。

**功能要求**：

1. **接收下单指令**
   - 提供 REST API 接收 TradingAgents 的下单请求（股票代码、方向、价格、数量）
   - 请求队列，防止重复下单

2. **实时行情监控**
   - 对接 A 股行情源（如通达信/同花顺行情推送）
   - 持仓股票实时价格跟踪
   - 盘口五档行情

3. **毫秒级下单执行**
   - 对接券商交易接口（如华泰/中信/同花顺量化 API）
   - 支持限价单、市价单、条件单
   - 订单状态跟踪（已报/部成/全成/撤单）

4. **实时风控止损**
   - 价格止损：跌破止损价自动触发卖出
   - 时间止损：持有超时自动平仓
   - 总仓位控制：单票仓位不超过 N%
   - 日内亏损限额：当日总亏损超过阈值暂停交易

5. **消息通知**
   - 企业微信 Webhook
   - QQ 机器人
   - 飞书 Webhook
   - 通知内容：开仓/平仓/止损/预警

**技术栈建议**：
- Python / Go（追求低延迟用 Go）
- WebSocket 行情推送
- SQLite / Redis 本地状态存储
- 与 TradingAgents 通过 REST API + Webhook 通信

**优先级**：待定（先完成股票筛选和持仓预警后再启动）
